  • Wähle ein Design (Theme) aus – es gibt kostenlose und kostenpflichtige

🚚 Schritt 5: Versand einstellen

  • Versandzonen & Kosten definieren (z. B. „innerhalb Deutschland: 4,99 €“)

  • Auch Selbstabholung oder kostenlose Lieferung möglich

  • Sobald alles eingerichtet ist, klickst du auf „Veröffentlichen“ – und der Shop ist online!

Damit kannst du z. B. www.maisoniris.de verwenden statt github.io.

So geht’s:

  1. Kaufe eine Domain bei z. B. IONOS, Strato, GoDaddy oder Netcup

  2. In den Domain-Einstellungen: CNAME-Eintrag setzen auf:

    lua
    deinbenutzername.github.io

  3. Im GitHub-Repository:

    • Datei namens CNAME anlegen (im Root-Verzeichnis)

    • Inhalt: www.maisoniris.de

  4. GitHub Pages erkennt die Domain automatisch

🔗 GitHub-Doku: Custom Domain Setup

🟢 Option 2: Benutzername aus Dateipfaden entfernen

Wenn du meinst, dass dein Benutzername z. B. im HTML oder in Bild-URLs auftaucht, dann kannst du ihn einfach manuell im Code ändern.

Zum Beispiel:

html
<!-- vorher -->
<img src="https://github.com/deinbenutzername/pfad-zum-bild.jpg" />

<!-- besser -->
<img src="https://example.com/meinbild.jpg" />

Oder du nutzt Bilderdienste wie Imgur oder bindest Bilder direkt von deiner eigenen Domain ein (wenn du eine verwendest).

🔴 Kann man den GitHub-Namen „komplett verstecken“?

Nicht direkt – wenn du GitHub Pages nutzt ohne eigene Domain, ist dein Benutzername immer Teil der URL. Das ist aber völlig normal und wirkt nicht unprofessionell.

Beispiel:

  • https://lena-atelier.github.io/maison-iris/ wirkt völlig okay und vertrauenswürdig

✅ Fazit:

Wenn dich dein Benutzername in der URL stört → eigene Domain kaufen und verbinden.
Ansonsten kannst du den Namen im Code oder in Bildpfaden manuell rausnehmen, aber die Seiten-URL bleibt so, wie GitHub sie vorgibt.
